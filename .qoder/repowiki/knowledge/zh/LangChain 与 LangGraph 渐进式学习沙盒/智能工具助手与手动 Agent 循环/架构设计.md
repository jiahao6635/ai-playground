- 入口与编排：`main.py` 定义 `run_agent` 核心循环，负责维护消息历史、检测 `tool_calls`、执行工具并构造 `ToolMessage` 回传，形成 LLM-Tool-LLM 闭环。
- 工具层：`tools.py` 使用 `@tool` 装饰器将普通 Python 函数（天气、计算、知识库搜索、时间）转换为 LangChain 工具，利用 docstring 和类型注解自动生成描述与参数 schema。
- 数据模型：`models.py` 定义 Pydantic 模型用于结构化数据表示（如天气信息）。
- 依赖方向：主流程依赖 `common` 模块获取 LLM 实例与配置，依赖 `tools` 模块获取工具集合；`tools` 模块可选依赖前序模块 `04-knowledge-base` 实现知识库检索功能。
